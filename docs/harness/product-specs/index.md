# 產品規格索引

這份索引追蹤持久產品規格。每個主要產品介面或使用者旅程應建立一份獨立規格文件。

## 規格清單

| 規格 | 狀態 | 文件 | 備註 |
| --- | --- | --- | --- |
| 訓練記錄 MVP | 草案 | [workout-logging-mvp.md](./workout-logging-mvp.md) | 記錄 sessions、動作、組數、歷史回顧與 MVP API I/O。 |
| 教練檢視流程 | Backlog | 尚未建立 | 需要使用者角色與分享模型。 |
| AI 進度助理 | Backlog | 尚未建立 | 應先從可解釋摘要開始，再進入處方式計畫。 |

## 規格文件應包含

- 使用者問題。
- 目標使用者。
- 核心流程。
- 非目標。
- 驗收條件。
- 資料模型。
- UX 狀態。
- 流程圖。
- API I/O。
- 可靠性與隱私注意事項。

實作計畫請另外使用 `harness/templates/exec-plan.md`，不要把執行紀錄混進產品規格。
