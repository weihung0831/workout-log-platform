# 產品

最後更新：2026-05-22

## 產品意圖

Workout Log Platform 幫助使用者記錄訓練、理解進步，並決定下一步要做什麼，同時避免把每次訓練變成繁瑣輸入工作。

## 主要使用者

- 追蹤力量、訓練量與穩定性的個人訓練者。
- 需要檢視學員訓練內容與執行率的教練。
- 想快速沿用過去訓練內容進行記錄的回訪使用者。

## 核心流程

- 建立或重複一份訓練。
- 記錄動作、組數、次數、重量、休息、備註與主觀強度。
- 依動作、肌群、週期或課表區塊回顧歷史。
- 找出進步、停滯、缺席訓練與恢復問題。
- 匯出或分享摘要給教練。

具體 MVP 流程圖、API I/O 與資料模型請看
`docs/harness/product-specs/workout-logging-mvp.md`。

## 領域實體

- 使用者
- 運動員檔案
- 訓練動作
- 訓練 session
- 單組紀錄
- 課表
- 課表區塊
- 個人紀錄
- 身體數據
- 教練備註

## 產品原則

- 訓練中快速記錄比完整但繁瑣的記錄更重要。
- 歷史資料應該回答「發生了什麼變化？」，而不是逼使用者整理試算表。
- 預設值應尊重漸進超負荷、恢復與穩定性。
- 教練視圖應凸顯值得注意的變化，而不是丟出大量原始資料。
- 加入 AI 功能時，必須解釋推理理由並保留使用者控制權。
